## R1｜BST-LDM：3D MRI→PET 多源条件潜空间扩散（MICCAI 投稿）
**关键词**：Latent Diffusion / DVAE / 多尺度结构先验 / 表格缺失建模 / 门控交叉注意力

**问题（Why）**
- PET 成本高、真实队列中缺失普遍；MRI→PET 的结构到功能映射天然多解
- 临床表格条件异质且缺失，3D 扩散训练/采样成本高

**方法（How）**
- **DVAE 潜空间扩散**：先将 PET 压缩到紧凑 latent，在 latent 上做扩散，提高 3D 推理效率
- **MRI 结构先验**：预训练结构特征提取器输出多尺度先验，并门控注入 3D U-Net，强化解剖一致性
- **表格缺失建模**：将表格变量 token 化并显式建模缺失，通过门控交叉注意力稳定融合非影像条件
- **可复现流程**：训练—评估—消融脚本化，支持对比与 ablation

**结果（So what）**
- 在 ADNI 测试集上，PSNR / SSIM / MAE 均取得最优（并给出对强基线的增益幅度）
- 支持缺失 PET 补全，并可用于下游分析/任务迁移

**我负责（Contribution）**
- 主导潜空间扩散框架实现与实验验证，搭建可复现训练—评估—消融流程
- 组织对比实验与 ablation，定位增益来源，整理论文所需图表与结论

## R2｜MC-GAN：MRI+临床表格条件的 PET 合成（ARCAI 接收）
**关键词**：Conditional GAN / Cross-Attention / 全局上下文建模 / 多损失约束

**问题（Why）**
- 仅用 MRI 合成 PET 容易缺失功能信息；表格条件注入不稳定，控制性差

**方法（How）**
- 从 MRI 提取结构信息作为先验，提升结构一致性
- 使用门控交叉注意力融合影像特征与表格特征，增强可控性
- 引入全局上下文建模组件（如 ViT）提升整体一致性
- 结合感知损失与多尺度结构损失降低模糊、保留细节

**结果（So what）**
- 在基准对比中取得 PSNR/SSIM/MAE 的优势表现，并完成模块/损失消融验证

**我负责（Contribution）**
- 与团队协作完成生成模型实现、对比实验与消融，整理论文图表与结论
- 与扩散版本形成互补：GAN 提供强基线与快速生成方案

## R3｜AFF-GMamba：Graph Mamba 跨模态自适应融合用于 AD 进展评估（TMI）
**关键词**：Graph Mamba / Patch-as-Node / 多模态融合 / 自适应归一化（AdaFusion）/ 生成式补全

**问题（Why）**
- AD 评估中 PET 缺失与跨中心差异导致泛化困难
- 传统 GNN 易过平滑/压缩；多模态融合缺少有效“交互”

**方法（How）**
- **生成式补全**：利用生成模型补全 PET，缓解缺失模态
- **Patch 构图**：以 patch 为节点，按空间邻接构建图结构
- **Graph Mamba**：高效建模长程依赖关系
- **AdaFusion**：在融合阶段引入自适应归一化与门控，增强跨模态交互与鲁棒性

**结果（So what）**
- 在 ADNI/NACC 上多指标领先，AUC 表现稳定（并提供了模块替换与消融）

## R4｜TFRN：频域卷积 + 历史融合的区域径流预测（IEEE TGRS 接收）
**关键词**：Time Series / Frequency-domain Conv / Historical Fusion / LSTM+Transformer

**问题（Why）**
- 区域径流预测需要同时刻画短期波动与长期模式；跨区域差异大、泛化难

**方法（How）**
- 引入**时域+频域卷积**捕捉多尺度模式并缓解噪声影响
- 在 Transformer 中设计**历史信息自适应融合模块**，引入远期历史提升长期依赖建模
- 在 CAMELS 系列数据上验证 7-day runoff forecasting 的稳定性与泛化能力

**结果（So what）**
- 多数据集上带来 NSE/RMSE 的一致改善，并提供开源代码链接

## R5｜ISFusion-Attack：自动驾驶 VLM 的实例-场景融合对抗攻击（T-ITS 投稿）
**关键词**：VLM Security / Black-box Transfer / Instance-Scene Fusion / Mixed Traffic

**问题（Why）**
- 自动驾驶混合交通场景中，VLM 需要理解“实例 + 场景上下文”
- 现有攻击方法对混合交通的复杂交互与迁移性支持不足

**方法（How）**
- 三模块框架：
  1) **Modality Enhancement**：增强视觉/语言特征与语义不变 caption
  2) **Attention Distraction**：在实例级与场景级嵌入误导文本，引导注意力偏移
  3) **Instance-Scene Fusion**：融合实例/场景 embedding，并通过代理模型集成增强黑盒迁移
- 在合成（CARLA）与真实世界基准（LingoQA）验证，评估多种 ASR 相关指标

**结果（So what）**
- 在多目标 VLM 上取得更高的攻击成功率，揭示自动驾驶 VLM 的关键安全风险点

# 实习 Internship

## I1｜头颅 NCCT 危急值识别与多智能体辅助报告（深圳大数据研究院 × 医院）
**周期**：2025.06–2026.02  
**角色**：算法研究合作者（与临床团队协作）  
**关键词**：3D CNN / 两级级联 / Multi-Agent / 可解释证据链 / 近实时分诊

**业务目标（Why）**
- 急诊头颅 CT 量大，人工阅片压力大；需要高敏感、低延迟的危急值筛查与分型，并输出可审计解释，辅助临床决策

**数据与任务（What）**
- 多中心真实队列：原始 7,877 例，质控后 5,490 例
- 两级任务：
  - Task1：危急/非危急快速筛查（二分类）
  - Task2：危急值五类亚型精分（五分类）

**系统方案（How）**
- **两级级联框架**：轻量 3D CNN 先筛查，再触发深层网络精分（兼顾速度与精度）
- **多智能体协作链路**：CritiScan Agent → Classify Agent → Multi-Agent Report Team，形成“识别—解释—报告”闭环
- **可解释证据链**：关键切片定位 + HU 物理阈值驱动显著性增强（如出血 HU 区间）+ 几何量化（Midline Shift 等）

**效果（So what）**
- Task1 在验证/实验中达到高准确与高召回，并满足近实时推理延迟
- Task2 多尺度融合后获得最优的五分类性能
- Agent 层响应可满足临床分诊时效需求

**我负责（Contribution）**
- 数据清洗质控、标签一致性校验、可复现划分与评测流程
- 基于 PyTorch/MONAI 的训练与推理 pipeline（预处理、采样、阈值策略、误差分析）
- 组织对比实验与消融，输出核心指标与可解释性材料，支撑论文/项目交付
